# Business Rules — mcp-wiki-server

## BR-1: 경로 이탈(Path Traversal) 차단 [보안 필수]
- read 도구의 path 입력은 문서 루트 기준 상대 경로만 허용한다.
- 절대 경로 입력은 거부한다.
- 루트 기준으로 resolve한 실제 경로가 루트 디렉토리 밖이면 거부한다 (`..` 등).
- 위반 시 오류: "허용되지 않는 경로입니다" (실제 파일 시스템 구조를 노출하지 않음).

## BR-2: 마크다운 파일만 서비스
- 검색/읽기 대상은 `.md` 확장자 파일로 한정한다.
- read에 비-md 경로 요청 시 거부.

## BR-3: 검색어 검증
- query가 비어 있거나 공백만 있으면 오류: "검색어를 입력해 주세요".
- 토큰화 후 유효 키워드가 0개면 동일 처리.

## BR-4: 존재하지 않는 문서
- read에 존재하지 않는 경로 요청 시 오류: "문서를 찾을 수 없습니다: {path}".

## BR-5: 대소문자 무시 매칭
- 모든 키워드 매칭은 대소문자를 구분하지 않는다 (파일명, 본문 동일).

## BR-6: 다중 키워드 AND 매칭 (Q1=A)
- 공백으로 구분된 모든 키워드가 문서(파일명+본문)에 포함되어야 매칭.

## BR-7: snippet 상한 (Q3=B)
- 문서당 snippet 최대 3개. 섹션 내 키워드 출현 횟수 내림차순, 동률 시 문서 내 등장 순.

## BR-8: 결과 정렬 및 제한 (Q3=B, D-1)
- match_count 내림차순, 동률 시 path 오름차순.
- 최대 limit개 (기본 10, 호출자가 조정 가능).

## BR-9: 결과 없음 처리 (Q6=A)
- 매칭 0건 시 빈 results + message "매칭되는 문서가 없습니다." 반환. 오류가 아님.

## BR-10: 인코딩
- 파일은 UTF-8로 읽는다. 디코딩 실패 문자는 대체 문자(�)로 처리하고 서비스는 계속한다.

## BR-11: 파일 시스템 오류 처리
- 스캔 중 개별 파일 읽기 실패(권한 등)는 해당 파일만 건너뛰고 검색을 계속한다.
- 루트 디렉토리 자체가 없거나 접근 불가하면 서버 시작 시 명확한 오류로 즉시 실패한다.
